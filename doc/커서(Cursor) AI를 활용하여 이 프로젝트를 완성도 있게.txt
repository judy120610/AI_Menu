커서(Cursor) AI를 활용하여 이 프로젝트를 완성도 있게 구현하기 위한 **프롬프트용 상세 디스크립션(Specification)**입니다. 이 내용을 Cursor의 **Composer(Ctrl+I)** 또는 **Chat(Ctrl+L)**에 붙여넣으면 AI가 의도를 정확히 파악하고 코드를 작성해 줄 것입니다.

---

### 📝 Project Specification: 방학 메뉴 추천 서비스 (Streamlit)

#### 1. 개요 (Overview)
- **목적**: 사용자가 보유한 재료와 요구사항을 바탕으로 월~금 점심 식단(요일별 3개 후보)을 추천하고, 최종 선택된 메뉴의 레시피를 제공하는 서비스.
- **기술 스택**: Python, Streamlit, Pandas, OpenAI API (선택 사항).

#### 2. UI/UX 레이아웃 설계 (Layout)
- **상단부**: 
    - 서비스 타이틀 및 설명.
    - '재료 추가하기' 영역: '종류(Selectbox)'와 '재료명(Text Input)'을 입력받아 하단 그리드에 즉시 반영.
- **중앙부 (재료 및 요구사항 그리드)**:
    - 6개의 컬럼(`st.columns(6)`) 구성.
    - 1~5컬럼: 생선, 고기, 야채, 냉동, 기타 카테고리별 재료 버튼 배치.
    - **핵심 동작**: 재료 버튼 클릭 시 `session_state`에 저장되며, 선택된 버튼은 배경색이 변하거나 강조되어야 함 (토글 방식).
    - 6컬럼: '요구사항' 영역. 기본 체크박스(매운음식 X 등)와 사용자가 직접 입력하여 추가하는 '커스텀 요구사항' 기능 포함.
- **하단부**:
    - [메뉴 생성하기] 버튼 (Primary 스타일).

#### 3. 핵심 기능 및 비즈니스 로직 (Functionality)
- **상태 관리 (State Management)**:
    - 모든 입력 데이터(재료 리스트, 선택된 재료, 추가된 요구사항, 생성된 메뉴 후보)는 `st.session_state`를 통해 브라우저 새로고침 시에도 유지되어야 함.
- **메뉴 생성 엔진 (AI Logic)**:
    - 입력값: `선택된 재료 리스트` + `선택된 요구사항 리스트`.
    - 출력 포맷: 월~금요일별로 각각 3개의 서로 다른 스타일(한식, 중식, 양식 등) 메뉴 리스트.
    - (구현 팁: OpenAI API 연동 코드를 작성하되, 테스트를 위해 가상의 Mock 데이터를 반환하는 함수를 먼저 구현할 것).
- **요일별 메뉴 선택 및 확정**:
    - 추천된 메뉴 후보(요일별 3개)를 라디오 버튼 형태로 노출.
    - [메뉴 확정하기] 클릭 시 선택된 5개의 메뉴를 '이번 주 메뉴' 표로 요약 출력.
- **레시피 출력**:
    - 확정된 메뉴들에 대해 각각의 상세 레시피(재료, 조리법)를 `st.expander`를 통해 가독성 있게 노출.

#### 4. Cursor에게 주는 상세 지시 (Specific Instructions)
- **스타일링**: "이미지의 느낌을 살려 재료 버튼은 연한 녹색 배경을 기본으로 하고, 선택되었을 때는 더 진한 색상이나 테두리가 생기도록 스타일을 적용해줘."
- **인터랙션**: "사용자가 새로운 재료나 요구사항을 추가하면 페이지 전체가 리로드되지 않고 해당 영역만 자연스럽게 업데이트되도록 `st.rerun()`을 적절히 사용해줘."
- **에러 핸들링**: "재료를 하나도 선택하지 않고 [메뉴 생성하기]를 눌렀을 경우 경고 메시지를 띄워줘."
- **코드 구조**: "기능별로 함수화(예: `init_state()`, `render_sidebar()`, `generate_menu_logic()`)하여 유지보수가 쉽게 작성해줘."

### 💡 Cursor 활용 팁 (How to use)

1.  **초기 설정**: `requirements.txt`를 먼저 생성해 달라고 하세요.
    > "Streamlit, OpenAI, Pandas가 포함된 requirements.txt를 만들어줘."
2.  **레이아웃 구현**:
    > "상세 디스크립션을 바탕으로 main.py의 전체 레이아웃을 잡아줘. 특히 이미지처럼 6개의 컬럼으로 나뉜 재료 선택 그리드를 중점적으로 구현해줘."
3.  **동적 기능 추가**:
    > "재료 버튼을 눌렀을 때 선택 상태가 유지되고 색상이 변하는 로직을 Session State를 사용해서 구현해줘."
4.  **AI 연동**:
    > "OpenAI API를 연동해서 프롬프트를 작성해줘. 보유 재료와 요구사항을 참고해서 월~금 메뉴를 JSON 형태로 받아오는 로직이 필요해."

이 디스크립션을 Cursor에 입력하면 훨씬 더 정확하고 깔끔한 코드를 한 번에 얻으실 수 있습니다! 추가로 필요한 부분이 있으면 말씀해 주세요.
